## 刷机后恢复步骤

### 1. 基础环境
```bash
# WiFi 连上后在板子上操作
mount -o remount,rw /
amixer -c 0 cset numid=2 1          # 麦克风通路（如果使用板载麦克风）

# 安装 Python（从备份用 micromamba 安装）
# 或者直接用备份恢复 Python 环境
```

### 2. 恢复文件
```bash
# 推主程序
adb push 2_BoardFiles/opt/hud_navi/. /opt/hud_navi/

# 推 AI 模型
adb push 2_BoardFiles/data/yolo/. /data/yolo/
adb push 2_BoardFiles/data/whisper/. /data/whisper/

# 推 NPU 驱动
cp /usr/lib/librknnrt.so /usr/lib/librknnrt.so.bak
cp /opt/hud_navi/usr/lib/librknnrt.so /usr/lib/librknnrt.so   # 或 push 备份中的

# 推启动脚本
cp 2_BoardFiles/etc/init.d/S99hudnavi /etc/init.d/
```

### 3. USB 麦克风（如果有）
```bash
# voice_auth.py 和 voice_command.py 里已配置为 plughw:2,0
# 如果是板载麦克风，改回 plughw:0,0
```

### 4. 启动
```bash
cd /opt/hud_navi
PYTHONPATH=/opt/hud_navi PYTHONUNBUFFERED=1 \
  nohup /opt/miniforge3/bin/python3 main.py > /tmp/hud.log 2>&1 &

# 验证
curl http://10.203.69.x:8080/ping
```

## V3.0 相对 V1.0 的改进

### 语音模块
1. **VAD 时序修复**: 一步录音 10 秒，消除 VAD 断档导致的语音开头丢失
2. **AI 优先策略**: Whisper NPU 优先，模板匹配降级
3. **动态阈值**: 环境噪音校准，减少误触
4. **USB 麦克风支持**: plughw 自动采样率转换
5. **繁简转换**: zhconv 自动转简体
6. **TTS 播放**: mpg123 + Edge-TTS 降级方案
7. **录入修复**: 同线程共享 audio_lock，递增计数器
8. **JSON 原子写入**: 避免文件损坏

### 已知限制
1. HUD 屏幕: 需要 Buildroot SDK 编译的 SDL2（带 Wayland 后端）
2. Whisper Base 准确度: 短命令OK，长专有名词有偏差
3. gTTS 需要翻墙
