# 摄像头 AI 预警 — 问题排查与修复

## 问题现象

YOLO 模型推理正常，但始终检测不到任何车辆。摄像头画面中有明显小车，模型输出最大置信度仅 0.03。

## 排查过程（按时间顺序）

### 1. 模型怀疑 → 排除
- YOLOv5s INT8（8MB）：obj 置信度最高 0.035，0 检测
- YOLOv5s FP16（15MB 交付包）：输出 18000 个假检测，全是噪声
- **结论：不是模型问题**

### 2. 输入格式怀疑 → 排除
- BGR uint8 / RGB uint8 / float32 BGR / float32 RGB 四种格式
- 带 sigmoid / 不带 sigmoid 两种后处理
- **结论：不是输入格式问题**

### 3. 模型转换怀疑 → 排除
- 从 ONNX 重新转换为 YOLOv5n RKNN（4.9MB）
- 测试结果相同：obj 置信度最高 0.24
- **结论：不是模型转换问题**

### 4. 输出格式怀疑 → 部分修正
- YOLOv5s 输出 3 个尺度 tensor
- YOLOv5n 输出合并 `[1, 25200, 85]`，第 5 列已是 obj×class 合并分数
- 修正了解析方式，但仍 0 检测
- **结论：不是主要问题**

### 5. NMS 缺失 → 修正
- 同一目标被 25 次重复检测为 8 辆车
- 加上按类别 NMS 去重
- **结论：次要问题，已修正**

### 6. 图像方向怀疑 → ✅ 根因
- 测试翻转图像：垂直翻转后 obj 置信度从 0.24 飙到 0.76
- **根因：USB 摄像头物理安装方向颠倒了 180°**
- 所有模型都在看倒置的画面，自然认不出物体

## 最终解决方案

| 问题 | 解决方案 | 文件 |
|------|---------|------|
| 摄像头装反 | `np.flipud()` 垂直翻转 | camera_pulse.py |
| YOLOv5n 输出格式 | 解析 `[1,25200,85]` 合并输出 | camera_pulse.py |
| 重复检测 | 按类别 NMS 去重 | camera_pulse.py |
| 模型转换 | 用 RKNN-Toolkit2 从 ONNX 转 | WSL 转换脚本 |
| 进程保活 | 外层 while True + guard.sh 重启 | camera_pulse.py + guard.sh |
| 输入格式 | BGR uint8 直入，不归一化 | camera_pulse.py |

## 最终效果

```
[Pulse] ⚠ x1 (NPU467ms) 汽车:0.85
```

- 模型：YOLOv5n FP16（4.9MB）
- 检测：汽车置信度 0.82-0.85，连续 6 帧无崩
- NPU 推理：~500ms/帧
- 系统：0 崩溃，导航不受影响

## 关键经验

1. **摄像头物理安装方向至关重要**——图像翻转是 YOLO 无法识别的首要原因
2. **不要怀疑模型**——原生 COCO 预训练模型开箱即用，不需要微调
3. **逐步排查**：模型 → 输入格式 → 输出格式 → NMS → 图像方向
4. **相机每帧 ~50KB JPEG，不影响系统性能**
